## Conventions & notes for agents

- This is a TypeScript project; keep code typed and run the type-checker before committing.
- Run the linter before opening a pull request; fix issues rather than disabling rules.
- Add or update tests when changing behavior, and make sure the suite passes.

## Do / Don’t

**Do**

- Read this file and existing code before making changes.
- Run `npm run test` and ensure it passes before finishing.
- Run `npm run build` to confirm the project still compiles.

**Don't**

- Introduce unrelated changes or large refactors without being asked.
- Use a different package manager than `npm` or commit a foreign lockfile.
- Commit secrets, credentials, or generated build artifacts.
